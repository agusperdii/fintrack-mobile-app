## Deskripsi Struktur & Cara Kerja File

### 1. Entry Point & Configuration
- **lib/main.dart**: Titik masuk aplikasi. Menginisialisasi Service Locator (DI), memvalidasi status autentikasi, dan membungkus aplikasi dengan Provider untuk manajemen state global.
- **lib/core/**: Berisi konfigurasi API, tema global (SavaioTheme), dan utilitas pendukung (Service Locator, Parser).

### 2. Presentation Layer (UI)
- **lib/views/pages/**: Layar utama aplikasi (Dashboard, Profile, Login). Berinteraksi langsung dengan Controller untuk mendapatkan data dan mengirim aksi pengguna.
- **lib/views/components/**: Mengikuti pola Atomic Design:
    - **Atoms**: Komponen terkecil (Button, Avatar, Heading).
    - **Molecules**: Gabungan beberapa atoms (TransactionItem, NotificationCard).
    - **Organisms**: Blok UI kompleks (Header, WeeklyPulseChart).

### 3. Controller Layer (State Management)
- **lib/controllers/**: Mengelola logika bisnis UI dan state aplikasi menggunakan `ChangeNotifier`. 
    - Contoh: `FinanceController` memanggil Repository untuk mengambil data transaksi, menyimpannya dalam variabel state, dan memicu `notifyListeners()` agar UI diperbarui secara otomatis.

### 4. Domain & Repository Layer
- **lib/models/repositories/**: Bertindak sebagai mediator antara Controller dan Data Source. Repository memproses data mentah dari API/Source menjadi format yang siap digunakan oleh Controller.

### 5. Data Source Layer & External API (☁️)
- **lib/models/data_sources/**: Bertanggung jawab atas komunikasi eksternal.
    - **RemoteDataSource & OCRDataSource**: File utama yang berinteraksi langsung dengan **External API**. Mereka mengirim request (POST/GET) dan menerima data mentah dari server.
    - **MockData**: Menyediakan data simulasi untuk keperluan testing atau pengembangan offline.

### 6. Model & Entity Layer
- **lib/models/entities/**: Definisi struktur data (Plain Old Dart Objects) seperti `AppData`, `Transaction`, dan `OCRResult` yang digunakan di seluruh aplikasi untuk konsistensi tipe data.
